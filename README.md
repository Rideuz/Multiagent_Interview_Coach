# Multi-Agent Interview Coach

Система из нескольких AI-агентов для проведения технического интервью с «мышлением» перед ответом.

## Требования


## Установка

```bash
pip install -r requirements.txt
```

Создайте файл `.env` в корне проекта (или задайте переменные окружения):

```
MULEROUTER_API_KEY=sk-mr-your-key
MULEROUTER_BASE_URL=https://api.mulerouter.ai/v1
INTERVIEW_LLM_MODEL=qwen3-max
INTERVIEW_LOG_FILE=interview_log.json
```

## Запуск

```bash
python main.py
```

Введите ФИО кандидата (на русском), позицию, грейд, опыт. Затем ведите диалог. Для завершения введите: **«Стоп интервью»** или **«Давай фидбэк»**. Лог сохраняется в `interview_log.json`.

## Архитектура

- **Interviewer** — ведёт диалог, задаёт вопросы. Не проверяет факты и не принимает решение о найме.
- **Observer** — анализирует ответы кандидата «в кулуарах», проверяет факты, определяет офф-топик и галлюцинации, подсказывает Интервьюеру, куда вести беседу и как менять сложность вопросов.

Каждый ответ пользователю строится так: **Observer** (скрытая рефлексия) → **Interviewer** (видимое сообщение). В логе в `internal_thoughts` видно, как агенты «общаются» между собой.

## Формат лога (interview_log.json)

- `participant_name` — ФИО на русском
- `turns` — массив ходов: `turn_id`, `user_message`, `internal_thoughts`, `agent_visible_message`
- `final_feedback` — структурированный отчёт (Вердикт, Hard Skills, Soft Skills, Roadmap)

## Секретные сценарии

Для прогона сценария (например, из ТЗ) вводите вводные и реплики вручную по шагам. После команды «Стоп игра. Давай фидбэк.» лог будет сохранён.
